# VoiceCanvas AI 设计文档

## 1. 题目理解

本项目选择题目二“AI 语音绘图工具”。题目要求用户不能依赖鼠标或键盘，而是通过语音指令完成绘图创作。因此本项目没有把重点放在自由手绘，而是选择“流程图 / 结构图 / 简单示意图”这个更适合语音控制的方向。

VoiceCanvas AI 的核心目标是：让用户通过中文自然语言创建、修改、连接、管理结构化图形，并在系统无法确定意图时主动提示，而不是误操作。

## 2. 产品定位

VoiceCanvas AI 是一个纯前端 Web Demo，面向流程图和结构化示意图创作。

适合场景：

- 快速生成简单流程图
- 用语音调整图形颜色、位置、大小和文字
- 用对象编号精确操作画布对象
- 在演示环境中展示语音到绘图操作的完整闭环

暂不定位为完整设计软件，也不追求复杂自由曲线绘制。

## 3. 计划支持的指令能力

项目初始计划支持以下能力：

- 创建基础图形：圆形、矩形、菱形、文本
- 修改对象颜色、位置、大小
- 删除对象
- 连接两个对象
- 自动对象编号
- 歧义提示
- 撤销
- 语音输入
- 一句话生成流程图模板
- 提供项目 README 和演示说明

## 4. 最终实现的能力

最终版本在原计划基础上继续扩展了多项语音交互能力：

- Web Speech API 中文语音输入
- 连续语音模式
- 语音反馈播报
- 创建圆形、矩形、菱形、文本
- 自动对象编号
- 对象列表展示
- 修改颜色、位置、大小
- 修改对象文字标签
- 删除对象及相关箭头
- 箭头连接对象
- 撤销历史
- 清空画布
- 清空画布二次确认
- 歧义提示
- 复合指令拆解执行
- 空间查询，例如“最右边的图形”
- 语音帮助指令，例如“我能说什么”
- 画布状态查询，例如“画布里有什么”

- 复制对象

- 生成登录流程图模板

## 5. 核心交互设计

### 5.1 对象编号

每个画布元素创建后自动获得唯一编号，例如 A、B、C。用户后续可以通过编号精确操作对象：

- “把矩形 B 改成绿色”
- “连接 A 和 B”
- “删除圆形 A”

这样可以降低语音指令中的歧义。

### 5.2 歧义处理

如果用户说“删除那个图形”这类无法确定目标对象的指令，系统不会直接执行，而是提示用户使用对象编号重新说明。

### 5.3 多轮语音上下文

系统支持选中对象：

- “选中 A”
- “把它改成绿色”
- “删除它”

这里的“它”会被解析为当前选中的对象 ID。

### 5.4 安全确认

“清空画布”属于高风险操作，因此系统不会立即执行。用户需要继续说“确认清空”，或者说“取消”放弃。

## 6. 技术方案

### 6.1 前端框架

项目使用 Next.js、React、TypeScript 和 Tailwind CSS 构建。

### 6.2 语音能力

项目使用浏览器 Web Speech API：

- `SpeechRecognition`：中文语音识别
- `speechSynthesis`：语音反馈播报

语音识别和语音反馈在浏览器侧完成，便于评委本地运行。

### 6.3 画布渲染

画布使用 SVG 渲染。画布中的每个对象都存储为结构化状态：

- id
- type
- x / y
- width / height / radius
- color
- text
- fromId / toId

React state 是唯一数据源，SVG 画布和对象列表都读取同一份状态。

### 6.4 指令解析

普通绘图指令优先提交到 Next.js 服务端接口，由大模型将中文自然语言解析成结构化 `DrawCommand`。例如：

- create
- update
- delete
- connect
- duplicate
- auto_layout
- generate_template
- clarify

服务端接口只允许模型返回白名单内的动作和字段，前端仍然只执行校验后的结构化命令。大模型没有直接修改画布状态的权限，避免口语化指令被误执行成未知操作。

未配置 `OPENAI_API_KEY`、网络失败或模型返回不可用 JSON 时，系统会自动回退到本地规则解析器，保证比赛现场仍然可以稳定演示。

## 7. 开发过程

项目按照独立 PR 分步实现，每个 PR 只做一个功能点。这样可以体现持续开发过程，也方便评委查看每一步的功能演进。

关键 PR 包括：

- PR 1：初始化项目与基础页面布局
- PR 2：实现 SVG 画布和基础图形渲染
- PR 3：实现文本指令解析器
- PR 8：实现 Web Speech API 语音输入
- PR 12：支持复合指令拆解执行
- PR 14：支持语音反馈播报
- PR 17：支持画布状态语音查询
- PR 21：支持语音选中对象和代词指令
- PR 24：支持语音一键整理画布
- PR 25：支持导出 SVG 文件
- PR 27：支持通过大模型解析自然语言绘图指令

## 8. 未完成部分和原因

暂未完成：

- 复杂自由曲线绘制
- 复杂图形组合
- 多人协作
- 云端保存
- 更复杂的多轮语义推理和跨多句话任务规划
- 真实设计软件级图层系统

原因：

- 比赛 Demo 优先保证核心链路完整和稳定。
- 项目定位是结构化语音绘图工具，不是完整设计软件。
- 纯前端实现更适合提交和复现，也能降低运行门槛。

## 9. 总结

VoiceCanvas AI 完成了从“语音输入”到“大模型指令理解”再到“画布执行”和“结果反馈”的完整闭环。项目重点体现语音交互的准确性、容错性、响应速度和可演示性。
