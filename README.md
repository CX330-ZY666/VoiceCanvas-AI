# VoiceCanvas AI

VoiceCanvas AI 是一个面向流程图、结构图和简单示意图创作的 AI 语音绘图 Web Demo。项目会按照独立 PR 逐步实现：先保证 `npm run dev` 可运行，再逐步加入 SVG 画布、文本指令解析、对象编号、对象修改、箭头连接、撤销、歧义提示、Web Speech API 语音输入和登录流程图模板。

## 当前进度

PR 6：实现箭头连接对象。

当前版本包含：

- Next.js 最小可运行项目结构
- 首页基础布局：顶部标题区、左侧控制区、中间画布、右侧对象列表、底部示例指令
- SVG 画布预览，尺寸基于 `900 x 600` viewBox
- 基础图形示例渲染：圆形、矩形、菱形、文本、箭头
- 画布元素 TypeScript 类型定义
- 本地文本指令解析器，可将常见中文指令转换为结构化 `DrawCommand`
- 左侧控制区支持手动输入指令并查看解析结果 JSON
- 画布对象存储在 React state 中，并自动分配 A、B、C 形式的对象编号
- 右侧对象列表从真实画布状态读取 ID、类型、颜色、位置和描述
- 文本指令可以创建基础对象、修改颜色、移动、缩放、删除对象和清空画布
- 文本指令可以用箭头连接两个已有对象
- 删除图形对象时，与它相关的箭头会一并移除
- README、Git 配置和基础工程配置

后续 PR 会继续接入撤销、歧义提示和语音输入。

## 技术栈

- Next.js：用于构建单页 Web Demo 和本地开发服务
- React：用于组件化页面与后续交互状态管理
- TypeScript：用于约束画布元素、指令解析结果等核心数据结构
- Tailwind CSS：用于页面布局和样式
- SVG：作为当前画布渲染方式，负责展示基础图形、文本标签和箭头
- Web Speech API：后续用于浏览器语音识别
- @phosphor-icons/react：用于按钮图标，提升控制区可读性

## 设计思路

项目采用“结构化画布状态 + 对象编号 + 空间关系解析 + 歧义确认”的方案。每个画布对象都会有唯一 ID，后续用户可以通过语音或文本精确引用对象，例如“把矩形 B 改成绿色”。

当前 PR 先定义基础画布元素结构，并用 SVG 渲染固定示例对象。后续 PR 会把这些固定示例替换为 React state 中的真实对象。

PR 3 增加本地规则解析器。解析器优先覆盖比赛演示所需的高频指令，例如创建图形、连接对象、修改颜色、移动、缩放、删除、清空、撤销和生成登录流程图模板。当前 PR 只展示解析结果，不直接修改画布状态。

PR 4 将画布对象提升为 React state，并提供对象编号工具。新增对象会从当前画布状态计算下一个可用 ID，右侧对象列表与 SVG 画布共享同一份元素数据。

PR 5 接入指令执行器。解析出的结构化指令会更新 React state，使 SVG 画布和对象列表同步变化。当前支持创建基础对象、按 ID 修改颜色、移动、缩放、删除对象和清空画布；箭头连接、撤销和模板生成仍按后续 PR 计划实现。

PR 6 接入箭头连接执行逻辑。用户可以说“连接 A 和 B”或“用箭头连接圆形 A 和矩形 B”，系统会为箭头分配新的对象 ID，并在 SVG 画布与对象列表中同步展示。
